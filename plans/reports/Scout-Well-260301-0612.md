## 9. Workflow: Webhook → Order → Subscription

```
PayOS Webhook
    ↓
payos-webhook/index.ts (184 lines)
    ├─ Verify HMAC signature (crypto.ts)
    ├─ Parse webhook data (PayOSWebhookData)
    └─ Call handlePayOSWebhook from webhook-pipeline.ts
        ↓
webhook-pipeline.ts (225 lines)
    ├─ Find order in DB (findOrder callback)
    ├─ Update order status (PENDING → PAID/FAILED)
    ├─ Find subscription intent (findSubscriptionIntent callback)
    └─ Activate subscription (activateSubscription callback)
        ↓
autonomous-webhook-handler.ts (236 lines)
    ├─ Validate state transition (isValidTransition)
    ├─ Update subscription_intents table
    └─ Audit log + mark complete
        ↓
billing-webhook-orchestrator.ts (120 lines)
    ├─ Resolve tenant from org ID
    ├─ Notify downstream systems
    └─ Return BillingOrchestrationResult
        ↓
vibe-subscription Engine
    ├─ Activate plan access
    ├─ Schedule renewal
    └─ Feature gate unlock
```
