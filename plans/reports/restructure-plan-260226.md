# Restructure Plan - WellNexus Locales & Store

## Context
The project `Well` has a very large `vi.ts` file (>117KB) and a monolithic `store/index.ts` that acts as a catch-all for state. This violates KISS and modularity principles.

## Objectives
1. Split `vi.ts` into domain-specific modules.
2. Decompose `store/index.ts` into appropriate slices.
3. Align with Binh Phap standards (modular, clean, type-safe).

## Phase 1: Locales Restructuring
- Create `src/locales/vi/` directory.
- Split `vi.ts` into:
    - `common.ts`
    - `auth.ts`
    - `dashboard.ts`
    - `wallet.ts`
    - `marketplace.ts`
    - `team.ts`
    - `referral.ts`
    - `copilot.ts`
    - `marketing.ts`
    - `health.ts`
    - `admin.ts`
- Update `src/locales/vi.ts` to export combined object.
- **Note:** Must sync with `en.ts` to avoid broken UI (Rule 8).

## Phase 2: Store Decomposition
- Create `src/store/slices/productSlice.ts`.
- Create `src/store/slices/marketingSlice.ts`.
- Move `products` and `fetchProducts` to `productSlice.ts`.
- Move `teamMembers`, `teamMetrics`, `teamInsights` to `teamSlice.ts`.
- Move `landingPageTemplates`, `userLandingPages` to `marketingSlice.ts`.
- Move `referrals`, `referralStats` to `teamSlice.ts` or a new `referralSlice.ts`.
- Move `quests` to `questSlice.ts`.
- Update `store/index.ts` to be a clean orchestrator.

## Phase 3: Verification
- Run `npm run build` to ensure no TS errors.
- Verify i18n keys are correctly resolved.
- Check store functionality in the app.
