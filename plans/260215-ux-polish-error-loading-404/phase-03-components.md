# Phase 03: Component Development (404 & Loading)

## Status: Pending
## Priority: Medium

## Overview
Develop reusable UI components for the 404 page and enhanced loading states.

## Requirements
- **NotFoundPage**: High-quality visual design (Binh Pháp style), clear call to action (Back to Home).
- **LoadingScreen**: Full-screen centered loading.
- **LoadingSkeleton**: (Optional) For content placeholders.

## Related Code Files
- `src/pages/NotFoundPage.tsx` (New)
- `src/components/ui/Loading.tsx` (New or Refactored)

## Implementation Steps
1. Create `src/pages/NotFoundPage.tsx` with Tailwind CSS.
2. Create or refactor loading components in `src/components/ui/`.
3. Test components in isolation (if possible) or via a test route.

## Success Criteria
- Responsive and visually appealing 404 page.
- Smooth loading transitions.
